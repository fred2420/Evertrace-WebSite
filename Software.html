Deploying Laravel on Azure

Step 1: Verify your PHP version and Install Composer

Click on your web app and go to the SETTINGS section and click on Application Settings, here you verify that the PHP version on your web app meets the minimum required by the version of Laravel you are running.

If not, change the PHP version and click the Save button to persist your changes. Then, you need to install composer which is the dependency manager for laravel. To do this, go to the DEVELOPMENT TOOLS section and click on Extensions. From there you click the Add button at the top left-hand corner.

This opens up a blade section from which you can then select Composer, agree to the Terms and Conditions, then click OK to install it.

Setting Up SQL Server to work with Laravel

  1. First off we need to install the driver for SQL server for PHP which can be downloaded below

    https://www.microsoft.com/en-nz/download/details.aspx?id=20098

  2. Running this exe will add a load of dlls into a folder of your choice the folder dose not matter at this point Copy the two DLLs below

    php_pdo_sqlsrv_7_nts_x64.dll

    php_sqlsrv_7_ts_x64.dll

  3. Paste these dll in C:\Bitnami\wampstack-7.1.22-0\php\ext

  4. Open the PHP ini file one level up at C:\Bitnami\wampstack-7.1.22-0\php

  5. Paste names of the dlls into the ini file

  6. Restart Wamp Server

  7. One more configuration copy the below code, you may need to change the config/database.php and add the following or it may be already there

    'sqlsrv' => [

    'driver' => 'sqlsrv',

    'host' => env('DB_HOST', 'localhost'),

    'database' => env('DB_DATABASE', 'forge'),

    'username' => env('DB_USERNAME', 'forge'),

    'password' => env('DB_PASSWORD', ''),

    'charset' => 'utf8',

    'prefix' => '',

    ],

  8. Open the .env file on the top level of the laravel project and change the conection string of the DB

    DB_CONNECTION=sqlsrv

    DB_HOST=traceone.database.windows.net

    DB_PORT=1433

    DB_DATABASE=TraceOneTest

    DB_USERNAME=Steve

    DB_PASSWORD=********

Including JSON files on Azure

If you upload a .json file to your Windows Azure website and try to access it, it would give you a 404 File Not Found error, because the MIME Type of .json is not set by default. This also applies in general to any file that might need a specific MIME Type. To fix this issue, FTP into your website and upload the following Web.config file which will set the correct MIME types. If you already have a Web.config file in place, just add the below to the appropriate section.

< configuration>

< system.webServer>

< staticContent>

< mimeMap fileExtension=".json" mimeType="application/json" />

< / staticContent >

< / system.webServer>

< / configuration>
